# Landing Page Builder — Feature Benchmark (summary)

Benchmarked against HubSpot, Unbounce, Instapage, Salesforce/Pardot, ActiveCampaign,
Mailchimp, Leadpages, Zoho. Drives the Connect Reach → Landing Pages UI build.

Key capability areas implemented in the UI:
1. List/management: grid + list, KPI tiles, status lifecycle (draft/published/scheduled/
   unpublished/archived + "unpublished changes"), filters, folders + campaign grouping,
   search/sort, row + bulk actions, templates gallery, AI generate.
2. Builder: section→column→block canvas, full block catalog, drag/reorder/duplicate/delete,
   undo/redo, layers tree, device toggle, global sections, save-state, inspector.
3. Styling: per-block + global theme (primary color, font, radius, width), section
   backgrounds (color/gradient/image + overlay), per-device visibility.
4. Forms: field types, progressive profiling, multi-step, reCAPTCHA, CRM mapping,
   submit actions (message/thank-you/redirect), sequence enrollment, consent/GDPR.
5. SEO/settings: title, meta, slug, domain, canonical, OG image, noindex, language,
   tracking (GA/pixels), custom head/footer code, cookie banner, password, expiry.
6. A/B + personalization: variants w/ traffic weight, goal, winner, running/ended.
7. Publishing: domain/slug, schedule, unpublish, revision history, device + live preview,
   share preview link.
8. Analytics: views, unique, submissions, conversion, bounce, avg time, daily trend,
   traffic sources, device split, submissions list, funnel, A/B results.
9. AI: generate page from prompt, copy assist, optimization score checklist (Leadmeter-style).
10. Templates: categorized gallery, start blank, save as template.

Provider names deliberately excluded from UI per project memory.
